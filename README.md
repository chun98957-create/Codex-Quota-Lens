# Codex-Quota-Lens
方案采用本地优先桌面应用：监听 Codex 本地会话事件，提取 token、额度百分比和重置时间，计算滚动消耗速度、最快使用时段，并通过个人历史数据模拟不同模型和推理强度。 几个关键设计： 区分“客户端报告值”“本地计算值”“预测值”。 不读取或保存 prompt、代码、推理及工具输出。 分别建模额度百分比与 credits，避免错误换算。 不假设“高推理固定消耗两倍”，而是通过个人历史数据校准。 模型和费率使用带版本的动态目录，不写死在业务代码中。 官方资料显示，Codex 消耗取决于任务规模、复杂度、模型和运行方式，并可能与其他 agentic 功能共享额度池；当前费率也按 input、cached input 和 output token 分别计算
