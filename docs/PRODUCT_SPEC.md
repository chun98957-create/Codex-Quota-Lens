# 产品规格

## 1. 产品定位

Codex Quota Lens 是面向 Codex 重度用户的本地额度助手。它不只是展示一个百分比，而是把“额度”转化为时间预算与决策建议。

一句话价值主张：**让用户知道额度花到哪里、会在何时用完、下一步怎样用更合适。**

## 2. 目标用户

### 个人开发者

- 使用 Plus/Pro 等个人方案
- 同时进行多个编码任务
- 经常在额度重置前耗尽，或在重置时仍剩大量额度

### 团队开发者

- 使用 Business/Enterprise/Edu
- 希望分析项目、时段和模型的消耗结构
- 可能具备 Enterprise Analytics/Compliance API 权限

### 不服务的用户

- 需要财务级精确账单对账的组织
- 希望绕过或规避服务限制的用户
- 希望把完整 prompt/代码上传给第三方分析的用户

## 3. 用户故事与验收标准

### US-01 查看实时额度

作为用户，我希望打开应用后 2 秒内看到最新额度与重置时间。

验收标准：

- 显示 used/remaining percent、window、reset time、last updated
- 新事件出现后 2 秒内更新
- 数据超过 60 秒未更新时显示“可能过期”
- 没有数据时给出可执行的诊断，而不是显示 0

### US-02 找到最快消耗时段

作为用户，我希望知道最近 7/30 天中哪些时间段额度下降最快。

验收标准：

- 提供 5 分钟和 1 小时粒度
- Top 时段至少包含时间、额度变化、token 构成、持续时间
- 支持按项目和模型过滤
- 跨重置窗口的数据不会错误相减

### US-03 判断能否撑到重置

作为用户，我希望看到当前速度与允许速度的差异。

验收标准：

- 给出 ETA 与重置时间对比
- 用绿色/黄色/红色表达风险
- 样本不足时显示宽区间和低置信度
- 零消耗或刚重置时不产生无穷大/错误警告

### US-04 模拟模型与推理强度

作为用户，我希望输入未来任务计划并比较多种方案。

验收标准：

- 可选择模型、推理强度、每日任务量和任务类型
- 展示预计消耗区间，而非单点假精度
- 显示费率版本与历史样本量
- 费率过期或模型未知时禁用“高置信度”标签

## 4. 信息架构

### Overview

- Hero：剩余额度 + 距重置时间
- Pace：实际 burn rate / 建议 burn rate
- Forecast：P50/P90 耗尽时间
- Live tokens：四类 token 的实时堆叠面积图
- Insight：一句可执行建议

### Speed

- 时间范围：24h / 7d / 30d / custom
- 热力图：星期 × 小时
- Top 10 高速窗口
- breakdown：项目、模型、推理强度、任务类型
- anomaly：超过个人基线 P95 的区间

### Planner

- 目标：坚持到重置 / 预留 X% / 限制每天额度
- Workload：简单修改、常规编码、复杂调试、长任务、自定义
- Scenario：model、reasoning effort、speed mode、tasks/day
- Output：预计额度、可完成任务数、风险、推荐组合

### Settings

- 本地目录发现与权限
- 数据源和 schema 版本
- 费率来源、更新时间和手动覆盖
- 保留周期、匿名化、导出、彻底删除

## 5. 关键文案

建议采用能区分事实与预测的文案：

- “客户端报告：已使用 63%”
- “过去 60 分钟：每小时 8.4 个百分点”
- “按当前速度，预计在 21:30–23:10 耗尽”
- “要坚持到周五 10:00，建议控制在每小时 3.1 个百分点以内”
- “此方案基于 14 个相似任务，置信度：中等”

避免：

- “保证还能使用 27 次”
- “高推理固定消耗 2 倍”
- “这是你的官方账单”

## 6. 风险与边界

- 本地事件 schema 可能变化：使用适配器、fixture 和 schema 健康检查。
- 额度可能被其他 agentic 产品消耗：检测“无本地任务但额度下降”的外部消耗段。
- 模型/费率动态变化：带版本远程目录 + 内置最后已知版本 + 用户覆盖。
- 个人额度与 credits 不是同一概念：分别建模，禁止把百分比直接换算成 credits，除非有可验证映射。
- 推理强度不是固定倍率：基于用户历史条件分布估计。

## 7. 成功指标

- 数据源首次连接成功率 ≥ 90%
- 新事件到 Dashboard 的 P95 延迟 < 2 秒
- 额度重置边界识别准确率 ≥ 99%
- 已有 20 个相似任务时，任务消耗 P50 预测 MAPE < 25%
- 用户在 30 秒内可以回答“能否撑到重置”
